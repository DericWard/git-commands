Initial set up of GIT:

  1. git --version - will show whether GIT is installed, if not Install GIT* then continue with this list.
  2. git config --global user.name "yourname" - as on GitHub without spaces between fore and surnames".
  3. git config --global user.email "your.email" - the email address you used for your GitHub account.
  4. git --global color.ui auto - automatically color different UI elements.

Project/repo directory set up:

  1. mkdir 'directory/repo name' - creates a directory/repo
  2. cd 'directory/repo name' - move into the newly created directory
  3. git init . - initialise GIT in this directory and its sub-directories.
  • rm -rf .git - remove a directory/repo from git (disable git in this repo)

Once you have created, and perhaps added content to, your files you move to the staging stage of the GIT workflow:

  1. git status - shows you which files have been added for staging and those which are not.
  2. git add 'file name(s)' - adds files for staging - multiple files require a space between each file name
  3. git add . - adds ALL files in this directory and its sub-directories for staging.

    4. Note: Adding all files with 'git add .' only works 'downwards' through a directory tree/structure - if you are working in a sub-directory and issue a 'git add .' command it will NOT add files that are 'above' the directory you are working in. E.g. files located in any parent directories will NOT be added to staging. To overcome this:

  4. git add . -A - this will stage files from the current directory and its child directories as well as from its parent directory.
  • git rm --cached 'filename(s)' - removes files from staging, if you've made a mistake or changes.
  • git rm -r --cached . - removes ALL currently staged files from staging.

When you've staged all the files you want, move to the commit stage of the GIT workflow:

  1. git commit -m "message - why you are performing this commit" - sends the staged file(s) for commit

    2. Note: Amend the commit message, if required, by issuing the following command:

  2. git commit --amend -m "your amended message" - this will replace the original message.
  • git restore 'filename' - removes the file 'filename' from the commit.
